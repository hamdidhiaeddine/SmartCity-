===========================================
POURQUOI L'ERREUR "No executable specified" ?
===========================================

CAUSE DU PROBLÈME:
==================

Qt Creator ne sait PAS automatiquement où se trouve votre exécutable
après la compilation. Il faut lui indiquer manuellement le chemin.

POURQUOI ÇA ARRIVE:
==================

1. CHEMIN DE BUILD VARIABLE
   → Qt Creator peut compiler dans différents dossiers selon le kit utilisé
   → Exemple: 
     - Qt 6.7.3 → build/Desktop_Qt_6_7_3_MinGW_64_bit-Debug/
     - Qt 5.9.9 → build/Desktop_Qt_5_9_9_MinGW_32bit-Debug/
     - Autre kit → build/win32/mingw-Debug/
   
   → Qt Creator ne devine pas quel chemin vous utilisez

2. CONFIGURATION NON ENREGISTRÉE
   → Le fichier .pro.user (configuration Qt Creator) peut être:
     - Corrompu
     - Incomplet
     - Utiliser un ancien format (Qt 4 au lieu de Qt 6)
     - Pointer vers un mauvais chemin

3. MULTIPLES CONFIGURATIONS
   → Si vous avez plusieurs kits Qt installés:
     - Qt 5.x
     - Qt 6.x
     - 32-bit / 64-bit
   
   → Qt Creator peut se tromper de configuration

4. PROJET DÉPLACÉ
   → Si vous avez déplacé le projet d'un dossier à un autre:
     - Ancien chemin: C:/Users/abc/Desktop/projet/
     - Nouveau chemin: C:/ayooo/
   
   → Les anciens chemins restent dans la configuration

===========================================
SOLUTION:
===========================================

Il faut CONFIGURER MANUELLEMENT le chemin de l'exécutable
dans Qt Creator une seule fois. Ensuite, Qt Creator s'en souviendra.

ÉTAPES:
1. Ouvrir la vue "Projects" (Ctrl+5)
2. Cliquer sur l'onglet "Run"
3. Cliquer sur "..." à côté de "Executable"
4. Naviguer vers: C:\ayooo\build\Desktop_Qt_6_7_3_MinGW_64_bit-Debug\debug\
5. Sélectionner: Atelier_Connexion.exe
6. Cliquer sur "Apply"

Après ça, Qt Creator saura toujours où trouver l'exécutable!

===========================================
POURQUOI L'EXÉCUTABLE EXISTE DÉJÀ?
===========================================

L'exécutable EST compilé et existe bien à:
C:\ayooo\build\Desktop_Qt_6_7_3_MinGW_64_bit-Debug\debug\Atelier_Connexion.exe

Le problème n'est PAS que l'exécutable n'existe pas.
Le problème est que Qt Creator ne SAIT PAS où il se trouve.

C'est comme si vous aviez un livre dans votre bibliothèque,
mais vous ne savez pas sur quelle étagère il est.
Le livre existe, il faut juste lui dire où le chercher!

===========================================
RÉSUMÉ:
===========================================

❌ ERREUR: "No executable specified"
✅ CAUSE: Qt Creator ne connaît pas le chemin de l'exécutable
✅ SOLUTION: Configurer le chemin une fois dans Qt Creator
✅ RÉSULTAT: L'application se lancera automatiquement ensuite
